## Why this module exists and how it works

Travel booking URLs are fundamentally unreliable when generated by LLMs:

- **Airlines**: booking URLs are session-dependent (contain tokens that expire in minutes)
- **Google Flights**: switched to base64-encoded protobuf URLs specifically to block programmatic access
- **Aggregators**: Kayak, Expedia, etc. use JavaScript-rendered pages; a raw HTTP fetch returns a shell, not prices
- **Bot detection**: Cloudflare, PerimeterX, and similar systems block automated fetches and return challenge pages

`generateTrip()` addresses this with a two-step approach:

**Step 1 — IATA resolution**: The agent uses `web_search` *only* to find the IATA airport codes for the origin and destination. This is a simple factual lookup that search handles reliably.

**Step 2 — Deterministic deep links**: URLs are constructed from **fixed templates** using documented stable formats from Kayak, Google Flights, Booking.com, and Google Hotels. The LLM fills in only `[ORIGIN_IATA]`, `[DEST_IATA]`, `[DATE]`, and `[CITY]` — it never generates URL structure. A `verify_url` client-side tool then confirms each link resolves to a real page (not a 404 or homepage redirect).

**What this guarantees:**
- Every link is a real, working page the user can click
- Prices shown are live and current at click time (not fabricated or stale)
- No booking URLs are invented, guessed, or constructed from model knowledge

**What this does NOT guarantee:**
- Locked-in fares (prices change between research and booking)
- Specific flight availability for the stated dates

## Upgrading to locked fares (Duffel API)

For real locked fares with actual checkout session URLs, integrate the Duffel API:

1. Sign up at duffel.com (free sandbox mode available)
2. Add `DUFFEL_ACCESS_TOKEN` to green's `.env`
3. Use the `flights-mcp` MCP server (github.com/ravinahp/flights-mcp, 169 stars) or call Duffel directly:
   - `POST /air/offer_requests` — search for flights with real pricing
   - `POST /air/sessions` — create a `links.duffel.com/...` checkout URL
   - The session URL is a real, verified checkout page — not a search result

For hotels with guaranteed availability, evaluate LiteAPI (liteapi.travel) — MCP-native, 3M+ properties, sub-second responses.

## Default origin

The default origin zip code is 22101 (McLean, VA → DCA/IAD). Users can override it with `/trip -default <zip>`. The saved default is stored in `default-origin` in this project directory.

## Command usage

```
/trip <destination zip>           — plan from default origin to destination
/trip <dest zip> -d YYYYMMDD      — plan for a specific travel date
/trip -default <zip>              — set a new default origin zip code
```

If no date is given, the agent defaults to two weeks from today with a 3-day stay.

## URL templates used (stable, documented)

```
FLIGHTS (Kayak):
  https://www.kayak.com/flights/[ORIGIN]-[DEST]/[YYYY-MM-DD]

FLIGHTS (Google):
  https://www.google.com/travel/flights?q=flights+from+[ORIGIN]+to+[DEST]+on+[DATE]

HOTELS (Booking.com):
  https://www.booking.com/searchresults.html?checkin=[DATE]&checkout=[DATE]&ss=[CITY+ENCODED]&group_adults=2&no_rooms=1

HOTELS (Google):
  https://www.google.com/travel/hotels?q=hotels+in+[CITY+ENCODED]+[DATE+ENCODED]

RENTAL CARS (Kayak):
  https://www.kayak.com/cars/[DEST_IATA]/[YYYY-MM-DD]-0900/[YYYY-MM-DD]-1700
```
